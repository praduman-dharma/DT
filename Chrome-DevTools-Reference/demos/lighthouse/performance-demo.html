Lighthouse Audit Demo

Instructions: Open DevTools → Lighthouse tab → Click "Analyze page load"

How to Run Lighthouse Audit

  1. Open Chrome DevTools (F12)
  2. Click the Lighthouse tab
  3. Select audit categories (Performance, Accessibility, etc.)
  4. Choose device type (Mobile or Desktop)
  5. Click Analyze page load
  6. Wait 30-60 seconds for audit to complete
  7. Review detailed report with scores and recommendations

Core Web Vitals (Key Performance Metrics)

Largest Contentful Paint (LCP)
When largest visible element renders. Target: < 2.5 seconds
First Input Delay (FID)
How fast page responds to user interaction. Target: < 100ms
Cumulative Layout Shift (CLS)
Unexpected layout changes. Target: < 0.1

Lighthouse Scoring

Optimization Results

After running Lighthouse audit, you'll see:

  • Overall Score: 0-100 (green: 90+, orange: 50-89, red: 0-49)
  • Opportunities: Changes that improve score (with estimated impact)
  • Diagnostics: Additional information and metrics
  • Passed Audits: What you're doing well
  • Detailed Metrics: LCP, FID, CLS, FCP, TTI, Speed Index

Common Issues Found by Lighthouse

Performance: Unoptimized images, unused code, large bundles, render-blocking resources
Accessibility: Low contrast text, missing alt text, unlabeled form inputs, improper heading order
Best Practices: Using HTTPS, password field types, front-end performance warnings
SEO: Missing meta tags, non-responsive design, robots.txt issues

Using Lighthouse Results

  1. Review Score: Understand which categories need improvement
  2. Prioritize: Focus on highest-impact opportunities
  3. Make Changes: Implement recommended fixes
  4. Re-audit: Run Lighthouse again to verify improvements
  5. Monitor: Run regularly to prevent regression

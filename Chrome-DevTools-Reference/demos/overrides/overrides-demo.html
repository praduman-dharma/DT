Overrides: Local File Testing Without Deployment

What Are Overrides?

Overrides allow you to modify local copies of files and have them persisted across page reloads, without deploying to the server.

Use Cases:
  • Test CSS/JavaScript changes without deploying
  • Mock API responses for testing
  • Test different user states
  • Experiment with design changes
  • Debug issues before committing code

Setting Up Overrides (Step-by-Step)

  1. Open DevTools (F12)
  2. Click Sources tab
  3. On left sidebar, click Overrides (may need to scroll)
  4. Click "+ Select folder for overrides"
  5. Choose or create a local folder
  6. Click Allow when browser asks for permission
  7. Start editing files in Sources panel or directly in folder

Folder Structure for Overrides

override-folder/ ├─ example.com/ │ ├─ index.html │ ├─ assets/ │ │ ├─ style.css │ │ └─ script.js │ └─ config.js ├─ cdn.example.com/ │ └─ library.min.js

Files are stored by domain, mirroring the URL structure of the website.

Workflow Example: Testing a Bug Fix

  1. 1. Create Override:
    Create file: override-folder/example.com/js/app.js
  2. 2. Copy Original Code: Copy original file contents
  3. 3. Apply Fix: Edit the code in override folder
  4. 4. Test: Reload page, test the fix
  5. 5. Verify: Test all affected features
  6. 6. Deploy: Copy fix to source, deploy to production

Verifying Overrides Are Active

  • Sources Panel: File shows override indicator/badge
  • Overrides Tab: Shows list of active overrides
  • File System: Check override folder for files
  • Timestamp: Override files have recent modification times

Clearing Overrides

Remove Specific Override: Delete file from override folder and reload
Remove All Overrides: Overrides tab → Click menu → Delete all overrides

Best Practices

  • ✓ Document: Note what each override does and why
  • ✓ Temporary: Keep overrides as temporary testing tools
  • ✓ Version Control: Git-ignore override folder (contains local changes)
  • ✓ Test Thoroughly: Test all code paths before deploying
  • ✓ Clean Up: Remove overrides when done testing
  • ✗ Don't: Leave permanent overrides in use
